<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Human input -> config/microclaw.config.yaml | User pastes API key into gitignored config file |
| MicroClaw scheduled prompt -> Python sidecar | Scheduled task fires a bash command; the command string is set at registration time |
| pipeline.py output string -> Discord channel | Return value of run_digest() is posted verbatim via MicroClaw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | config/microclaw.config.yaml | mitigate | File is gitignored (confirmed in 01-01-PLAN); never commit; contains real API key after Task 1 substitution |
| T-04-02 | Information Disclosure | OpenRouter API key in logs | mitigate | pipeline.py passes key as parameter to filter.py; key is never printed to stdout or included in run status messages |
| T-04-03 | Tampering | MicroClaw scheduled task prompt | accept | Prompt string set once at registration; MicroClaw runs with sandbox.mode=off (confirmed in spike); attacker would need local filesystem access to modify |
| T-04-04 | Information Disclosure | Discord milestone posts | accept | Posts contain only run metadata (counts, run ID, schedule type) - no API keys, no feed content, no secrets |
| T-04-05 | Denial of Service | 08:00 schedule firing during OpenRouter outage | mitigate | pipeline.py outer except catches FilterError; run records status=failure; Discord failure message posted - no unhandled crash |
</threat_model>
