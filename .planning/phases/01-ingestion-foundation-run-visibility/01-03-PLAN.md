<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workflow/service layer -> run repository | Application logic creates durable run records and retry lineage |
| workflow/service layer -> event journal | Runtime milestones become append-only audit events |
| local operator/browser -> trace URL | Exact run traces are opened through the localhost MicroClaw surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | R | `runs/repository.py` | mitigate | Store `run_id`, `short_run_id`, `run_label`, and `parent_run_id` on every run so retries/reruns cannot repudiate which attempt produced an outcome. |
| T-01-08 | T | `runs/events.py` | mitigate | Provide insert/list APIs only and test that the event journal never exposes update/delete behavior, preserving append-only history. |
| T-01-09 | I | `build_trace_url` | mitigate | Default all trace URLs to `http://127.0.0.1:10961/runs/<run_id>/events` so run traces land on the exact local raw event/log view unless the operator explicitly changes `PAH_TRACE_BASE_URL`. |
</threat_model>
