<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| watchlist config -> fetch layer | Local config determines the only remote URLs the app may contact |
| remote feeds -> parser | Untrusted XML/Atom payloads cross into ingestion logic |
| ingestion service -> SQLite raw store | Parsed source metadata becomes durable local records |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | T | `feed_ingestor.py` | mitigate | Parse only allowlisted HTTPS URLs from `config/watchlist.yaml`, cap payload size to `1_048_576` bytes, and turn malformed payloads into explicit source failures. |
| T-01-05 | I | `feed_ingestor.py` | mitigate | Never accept runtime URL overrides and set `follow_redirects=False` so dynamic fetch targets cannot become SSRF pivots. |
| T-01-06 | R | `raw_items.py` | mitigate | Persist `run_id`, `source_id`, `source_tier`, and `source_kind` with every row so later visibility work can trace each candidate back to one approved source. |
</threat_model>
