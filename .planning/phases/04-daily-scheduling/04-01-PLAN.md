<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo config -> MicroClaw scheduler | `config/microclaw.config.yaml` controls cron timezone interpretation |
| Discord chat -> MicroClaw scheduler tool | `@Personal Agent Hub ...` prompt causes runtime schedule mutation |
| Windows Startup folder -> user session login | `.lnk` auto-launches MicroClaw on login |
| SQLite runtime DB -> scheduler poller | `scheduled_tasks` row persists scheduler state across process restart |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Duplicate schedule rows | mitigate | Always run duplicate-cancel guard before registration; verify exactly one active row after registration |
| T-04-02 | Denial of Service | Direct DB insertion bypasses in-memory scheduler | mitigate | Never write `scheduled_tasks` directly; register through MicroClaw @-mention only |
| T-04-03 | Elevation of Privilege | Startup shortcut written to all-users Startup folder | mitigate | Use `[Environment]::GetFolderPath("Startup")` only; do not use CommonStartup or service install |
| T-04-04 | Configuration Drift | Timezone hardcoded outside config | mitigate | Only set `timezone: "Asia/Jerusalem"` in YAML; grep ensures no Python timezone literal |
| T-04-05 | DoS | MicroClaw not running after reboot | mitigate | Install Startup shortcut and verify proxy restart persistence; Phase 5 later covers missed-run catch-up |
</threat_model>
