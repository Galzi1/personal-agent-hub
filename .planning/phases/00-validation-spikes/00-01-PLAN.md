The user must complete these steps that Claude cannot automate:

**1. Discord Bot Setup (required for Spike 1):**
- Go to https://discord.com/developers/applications
- Click "New Application", name it (e.g., "PersonalAgentHub-Dev")
- Go to Bot section, click "Reset Token", copy the token (save it securely)
- Under "Privileged Gateway Intents", enable "Message Content Intent"
- Go to OAuth2 > URL Generator, select scope "bot", permissions "Send Messages" and "Read Message History"
- Copy the generated URL, open it in browser, invite bot to a test Discord server
- Note the channel ID where the bot should post (right-click channel > Copy Channel ID, requires Developer Mode in Discord settings)

**2. Install MicroClaw (PowerShell):**
```powershell
iwr https://microclaw.ai/install.ps1 -UseBasicParsing | iex
```
After installation, run the setup wizard:
```powershell
microclaw setup
```
When prompted, enter:
- Discord bot token (from step 1)
- Ollama base URL: http://127.0.0.1:11434/v1 (default)
- Web panel: enabled (default port 10961)

Run diagnostics to verify installation:
```powershell
microclaw doctor
```

**3. Install Ollama (needed for MicroClaw's LLM provider):**
- Download from https://ollama.com/download/windows
- Run the installer
- Verify with: `ollama --version` (should show v0.20.6 or similar)

**4. Check system RAM (determines Ollama model choice for Plan 02):**
```powershell
systeminfo | findstr "Total Physical Memory"
```
- If >= 32GB: Qwen3 32B is feasible
- If 16-31GB: Gemma 4 27B MoE is the safer choice
- If < 16GB: Ollama spike may need to use smaller models

**5. Verify MicroClaw data directory is gitignored:**
Check what directory MicroClaw uses for data (shown during `microclaw setup` or `microclaw doctor`). Ensure that directory path is in `.gitignore`. If MicroClaw stores data inside the repo, add the path. Typical location is `~/.microclaw/` (outside repo) or a local `data/` directory.
