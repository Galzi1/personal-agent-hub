<task type="auto">
  <name>Task 1: Define memory and story tables in Prisma schema</name>
  <files>prisma/schema.prisma</files>
  <action>Add `memory` and `story` models to the Prisma schema. Include fields:
- `memory`: id (String @id), content (String), createdAt (DateTime @default(now()))
- `story`: id (String @id), title (String), content (String), confidence (Float), createdAt (DateTime @default(now()))
Establish a foreign key relationship if needed.
</action>
  <verify>
    <automated>rtk grep "model story" prisma/schema.prisma && rtk grep "model memory" prisma/schema.prisma</automated>
  </verify>
  <done>Prisma schema contains `memory` and `story` models with specified fields.</done>
</task>

<task type="auto">
  <name>Task 2: Implement deduplication pipeline</name>
  <files>src/story/dedup.ts</files>
  <action>Create a TypeScript module that:
- Accepts raw update objects (e.g., {id, source, content}).
- Groups updates by similarity (use simple Jaccard or cosine on embeddings).
- Generates a single canonical story per group with a confidence score (e.g., proportion of matching updates).
- Stores the result in the `story` table using Prisma client.
</action>
  <verify>
    <automated>npm test -- -t "deduplication"</automated>
  </verify>
  <done>Deduplication function produces a story record and persists it to the database.</done>
</task>

<task type="auto">
  <name>Task 3: Wire API endpoint to trigger deduplication</name>
  <files>src/api/story/route.ts</files>
  <action>Expose a POST endpoint `/api/story/dedup` that receives a batch of raw updates, calls the deduplication function, and returns the created story IDs.
Validate input schema using Zod.
</action>
  <verify>
    <automated>curl -X POST http://localhost:3000/api/story/dedup -H "Content-Type: application/json" -d "[{\"id\":\"u1\",\"source\":\"feed1\",\"content\":\"example\"}]" | grep \"storyId\"</automated>
  </verify>
  <done>POST endpoint creates canonical stories from raw updates and returns IDs.</done>
</task>
